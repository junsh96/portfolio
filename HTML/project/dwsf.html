동화 스마트 팩토리

개요

동화기업 인천 - 베트남 공장내 데이터와 그에 따른 위험감지.
내부 작업 지시등 공장내 필요한 기능들을 개발한 시스템입니다.



기여한 부분.

저는 주로 시설 파트를 담당하였습니다.

시설 파트는 통지 - 오더 - 작업으로 이어지는 요청 프로세스를 처리하는 영역이며.
최초 시설 작업자가 작업에관한 통지를 요청 - 관리자가 해당 요청을 수락합니다.
그와 함께 수락된 요청이 오더에 생성되게 되며.
해당 오더를 작업자가 완료하게 된다면 작업일지에 해당 오더가 생성되게됩니다.
마지막으로 관리자가 해당 작업이 완료된것을 확인한뒤 작업을 완료시키면 작업을 끝 마칠수있습니다.
완료된 작업 리스트는 관리자가 작업자들에게 엑셀 리스트를 생성하여 메일로 전송시킬수 있습니다.

이 과정들은 soap통신을 통해 이루어지며 만에하나 soap통신이 원할하지 않아 누락된 데이터건도 정상적으로 등록하기 위하여 배치가 돌아가게 되어있습니다.

트러블 슈팅

라이브 서버에 운영중 지속적으로 개발을 하다보니 실수가 커밋되어 운영서버에서 에러가 발생한적이 있습니다.
커밋된 직후 테스트에서 발견되어 바로 pm에게 보고하였고, pm이 고객측에 연락한뒤 빠르게 수정하여 에러를 잡았었습니다.
실수가 발생한 이유는 db내에서 사용되지 않는 컬럼들이 프로젝트에서 사용되어야 하는데 사용되지 않는 것으로 착각하였고,
실수를 했다 판단하여 수정하여서 커밋했습니다. 개발 서버에서는 에러가 발생하지 않아 괜찮을거라 생각했지만 운영서버의 배치에서 문제가 발생하였던 것이였습니다.
pm께서는 별다른 질책없이 다음부터는 더 조심하라고 말씀하시고 넘어가셨습니다.
해당 경험을 토대로 커밋을 하기전 테스트는 더더욱 꼼꼼히 해야하며, 실수를 했다고 판단을 했을때 저 스스로의 판단을 믿지 말고 보고부터 한뒤 판단해야한다는걸 깨달았습니다.
